2018-11-09T23:41:00+08:06/archives/overview/Overview of Maverick2021-01-03T02:05:20.438663+00:00AlanDecodehi@imalan.cnhttps://www.imalan.cn<h2>Overview</h2> <p>Maverick is a static blog generator built with python. Like Hexo and Jekyll, it takes Markdown(<code>.md</code>) files as input, and output beautifully formated and well structured website pages(<code>.html</code>). It has a lot built-in useful features(feed, search, sitemap, etc.), with extended Markdown syntax and enhanced image processing pipeline.</p><p>If you are tired of intricate plugins and complicated configurations, just give Maverick a try. You focus on writing, let Maverick take care of the rest.</p><p>Pull requests are surely welcome. If you have any questions or suggestions, please head to <a href="https://github.com/AlanDecode/Maverick/issues">issue area</a> and leave us a message. Before that, let's finish this document.</p><h2>Usage</h2> <p>Maverick is build with modern Python, currently at least <strong>Python 3.5</strong> is required, make sure you have it installed on your machine.</p><h3>Install</h3> <p>Clone this repository:</p><div class="highlight"><pre><span></span>git clone https://github.com/AlanDecode/Maverick.git ./Maveric <span class="o">&amp;&amp;</span> <span class="nb">cd</span> ./Maveric </pre></div> <p>Install all dependencies:</p><div class="highlight"><pre><span></span>pip install -r prod_req.txt </pre></div> <p>If error occurs, please verify your Python and pip version. Then edit the default configuration file:</p><div class="highlight"><pre><span></span>vi ./config.py </pre></div> <p>For now let's use the default settings. Type this command in your terminal:</p><div class="highlight"><pre><span></span>python ./build.py </pre></div> <p>...and a sample static site is generated in <code>test_dist</code> folder! You can then upload them to GitHub Pages or any other server.</p><h3>Generate your own site</h3> <p>By default, Maverick searches all <code>.md</code> files recursively under <code>test_src</code> folder, so put your Markdown files in it and run <code>python ./build.py</code> then everything you need will be in <code>test_dist</code> folder.
